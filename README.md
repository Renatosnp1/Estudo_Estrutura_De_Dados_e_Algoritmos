## Visão geral do projeto

Este repositório foi criado com o objetivo de **estudar algoritmos e estruturas de dados** por meio de problemas práticos da plataforma **LeetCode**.  
A ideia é registrar minha jornada de aprendizado, refletir sobre as soluções, comparar abordagens e consolidar conhecimentos em lógica de programação, análise de complexidade e boas práticas de código.

### Objetivos principais

- **Praticar algoritmos e estruturas de dados** usando exercícios reais do LeetCode.
- **Desenvolver raciocínio lógico e matemático** para resolução de problemas.
- **Aprimorar habilidades em linguagens de programação**, com foco inicial em Python (podendo incluir outras linguagens no futuro).
- **Documentar o processo de aprendizado**, registrando:
  - Problemas estudados.
  - Ideias de solução e variações.
  - Erros cometidos e como foram corrigidos.
  - Melhorias de desempenho (tempo e memória).
- **Criar um portfólio técnico público** que reflita minha evolução e demonstre meus conhecimentos para recrutadores e outros desenvolvedores.

### Tecnologias e temas abordados

- **Linguagens**: principalmente Python.
- **Estruturas de dados**:
  - Arrays, Strings, Listas Ligadas, Pilhas, Filas.
  - Árvores, Grafos, Heaps, Hash Tables, etc.
- **Paradigmas e técnicas**:
  - Recursão, Programação Dinâmica.
  - Busca (DFS, BFS), Backtracking.
  - Algoritmos gulosos (Greedy), Divide and Conquer.
- **Análise de complexidade**:
  - Notação \(O(n)\), \(O(\log n)\), \(O(n^2)\), etc.
  - Trade-offs entre tempo e memória.

## Organização do repositório

- **`/`**: scripts gerais e arquivos de configuração.
- **Pastas de problemas (futuras)**:
  - Organização sugerida por **plataforma / tópico / dificuldade**  
    (ex.: `leetcode/arrays/easy/`, `leetcode/graphs/medium/`, etc.).
- Cada problema poderá conter:
  - Arquivo da solução em código.
  - Comentários explicando a ideia principal do algoritmo.
  - (Opcional) Anotações adicionais em markdown com reflexões sobre o problema.

## Como estou utilizando este repositório

- Escolho problemas do LeetCode por tema ou dificuldade.
- Implemento uma ou mais soluções, buscando:
  - Versão inicial correta.
  - Possíveis otimizações (tempo/memória).
- Registro aqui no README (ou em arquivos auxiliares) **o que aprendi com cada problema**, para reforçar o entendimento e mostrar meu processo de raciocínio.
